Genie can give you a list of intelligently organized wishes which are in context for a given magicWord based on a priority calculated by a sorting algorithm. Let's say the magic word we've given genie to match is "At".
  1. * King of the Hill for the given magicWords (genie will optimistically anticipate this)
  2. * On Deck for the given magicWords (also optimistically anticipated)
  3. At - Equal - If the given magic word is equal to any magic words of a wish
  4. AtTask Rocks! - Starts With - If the given magic word is the start to any magic word of a wish
  5. Love AtTask! - Word Starts With - If the given magic word is the start to any word in a magic word
  6. Love cats - Contained - If the given magic word is contained in any magic words of a wish
  7. Alligator Time! - Acronym - If the given magic word is an acronym of any magic words of a wish
  8. Happily Ever After - Simple Match - If the given magic word matches the order of characters in any magic words of a wish.
Note: if two wishes match equally, then the one which was most recently registered is returned first. Also, because wishes can have an array of magic words, if two wishes match equally, but one's match is first in that array and the other is second in the array, the first one will win even if it was more recently registered.
* The first priority in the list is where genie learns the user as it is used. As wishes are made with a specific magic word, it becomes the "King of the Hill" for that magic word. If another wish is made with the same magic word, then it becomes "On Deck." If this on deck wish is made with that same magic word it will then become "King of the Hill" and the previous king becomes on deck.
As a final note, you can provide your own matching algorithm if you want to using genie.overrideMatchingAlgorithm...
