Context is everything, and genie understands this. If a wish is not in context then it will not be returned as a match using getMatchingWishes and it cannot be made using makeWish. In single page application, genie could be loaded up with a ton of wishes and suddenly you could have a "Send Message" wish when you're looking at your contacts in an email application (which would make no sense at all).
A context is a string. Genie's context is an array of strings. Wishes have a context property that looks like this: 
{
  all: ['context1', 'context2'],
  any: ['context3', 'context4'],
  none: ['context5', 'context6']
}

How to be in context

There are a few ways for a wish to definitely be in context:
  • Genie's current context is the default context (the default context is 'universe')
  • The wish's context.any contains the default context
  • The wish's context is equal to the current context
If none of these are true, then these things must be true for the wish to be in context:
  • Genie's context does not contain any of the wish's context.none contexts if it exists.
  • Genie's context contains at least one of the wish's context.any contexts if it exists.
  • Genie's context contains all of the wish's context.all contexts if it exists.

Context functions

Genie gives a few functions that help with context:
  • context(newContext) - Sets the current context (if newContext is provided) and returns the current context
  • addContext(context) - Adds the context(s) to genie's current context
  • removeContext(context) - Removes the context(s) to genie's current context
  • restoreContext - Sets and returns the current context to the default context
  • revertContext - Sets and returns the current context to the previous context
