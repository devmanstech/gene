Short Version

<a href="/home" genie-wish=>Go Home</a>
That is the equivalent to: 
genie({
  magicWords: 'Go Home',
  action: '/home'
});

Long Version

<a href="/home"
  genie-wish="Go Home"
  wish-id="go-home-wish-id"
  wish-context="comma separated,wish contexts"
  wish-data="scopeVariable"
  wish-event="click"
  event-modifiers="altKey"
  wish-i-icon="glyphicon glyphicon-envelope"
  wish-img-icon="/home-logo.png">Home</a>
That is the equivalent to: 
var wish = genie({
  id: 'go-home-wish-id',
  magicWords: 'Go Home',
  context: ['comma separated', 'wish contexts'],
  data: scopeVariable
  action: function(wish) {
    var event = new MouseEvent('click', {
      view: window,
      bubbles: true,
      cancelable: true,
      altKey: true,
    });
    // ux-genie adds the element property to data.uxGenie.
    wish.data.uxGenie.element.dispatchEvent(event);
  }
});
wish.data.uxGenie.iIcon = 'glyphicon glyphicon-envelope';
wish.data.uxGenie.imgIcon = '/home-logo.png';

Attributes

  • genie-wish - The directive itself. This can be a comma separated list for multiple magic words.
  • wish-id - The id of the wish
  • wish-context - A comma separated list of contexts
  • wish-data - A scope object to be assigned to the wish's data object. This is doubly bound.
  • wish-event - The dom event to trigger on the element. Defaults to 'click'.
  • event-modifiers - a comma separated list of modifiers to add to the event triggered on the element.
  • wish-i-icon - Used to set data.uxGenie.iIcon (see the ux-lamp directive for what this is used for).
  • wish-img-icon - Used to set data.uxGenie.imgIcon (see the ux-lamp directive for what this is used for).
If for some reason you would rather not have the genie-wish attribute be the magic words, you can leave that attribute empty and the genie-wish directive will look for a name, id, or the text of the element (in that order) and assign the magicWords to that value (split by commas). If none of these have values, you will get an error.
